01このアプリって何？

「会社名とURL」を一覧で貼り付けると、そこからメールアドレス・お問い合わせフォーム・特商法ページなどを自動で探してくれるアプリです。

手動でやると1社3〜5分かかる作業が、100社で5分以内に終わります。営業リスト作成の前処理を一気に自動化するやつ。

できること：
・公式サイトから info@〜 系メアドの抽出
・問い合わせフォームのURL抽出
・特商法・運営会社・プライバシーページも掘る
・CSV / Excel形式でダウンロード
できないこと：
・会社名リストを0から作ること（会社名とURLは自分で用意する必要あり）
・Bot対策が強いサイトの突破（Playwright等が必要）
・直接メール送信（これは別ツール）

02最初の1回だけやる準備

このアプリは初回起動時に必要なライブラリを自動でインストールします。2-3分かかります。

準備：Python が入っているか確認

1
Windowsの「スタート」 → 「PowerShell」を検索して開く（黒い画面が出る）
2
python --version と打ってEnter。「Python 3.9」以降の表示が出ればOK。
3
「そんなコマンドない」と言われたら python.org から最新版をダウンロード → インストール時に「Add Python to PATH」にチェックを忘れずに。
Macの人：Macには最初から Python が入っています。ターミナルで python3 --version。3.9以降ならOK。

03毎回の起動手順（2回目以降）

1
デスクトップの 営業リスト作成アプリ フォルダを開く
2
start.bat をダブルクリック
3
黒いウィンドウが開く。「営業リスト作成アプリ 起動中...」と表示される
4
「http://localhost:5000」が表示されたら、ブラウザでそのアドレスを開く（ブクマしとくと便利）
5
アプリ画面が出たら起動完了。ここから使える。
終わらせる時：黒いウィンドウで Ctrl+C を押す、または×ボタンで閉じる。
閉じないとPCのリソースを使い続けるので、使い終わったら必ず閉じて。

04入力の書き方

テキストボックスに「1行に1社」の形で、「会社名」と「URL」をカンマ（,）またはタブで区切って書きます。

基本形（2列）

リクルートホールディングス, https://recruit-holdings.com/ja/
パソナグループ, https://www.pasonagroup.co.jp/
ウィルグループ, https://willgroup.co.jp/

業種も入れたい場合（3列）

エン・ジャパン, https://corp.en-japan.com/, 求人媒体
マイナビ, https://www.mynavi.jp/company/, 求人媒体
ビズリーチ, https://www.visional.inc/ja/, HR Tech

コメント行

#で始まる行は無視されます。メモを書いておくのに便利。

# 2026年5月 営業対象リスト
# 人材系大手のみ
リクルートホールディングス, https://recruit-holdings.com/ja/
パーソルホールディングス, https://www.persol-group.co.jp/

CSVアップロード

既にCSVファイルがあるなら「📎 CSVアップロード」ボタンで読み込めます。1列目が会社名、2列目がURL、3列目が業種（任意）。

「デフォルト業種」入力欄を使うと、3列目を書いていない全社にまとめて業種を付けられる（例：全員が人材系なら「人材紹介」と入れておく）。
URLは https:// から書くのがおすすめ。省略しても動くけど、確実にするため。

05実行と進捗の見方

1
「▶ 収集開始」ボタンを押す
2
右上のバッジが「実行中...」オレンジ色に変わる
3
進捗バーが伸びていき、「3 / 30」のようにカウントが進む
4
結果テーブルに1社ずつリアルタイムで追加される
5
完了すると右上バッジが「完了」緑色になる → ダウンロードボタンが有効化

目安の時間

件数目安時間
10社約1分
50社約5分
100社約10分

デフォルトの「各社の間隔」は2秒。速くしたい時は1秒に下げられるけど、サーバー負荷がかかりやすくなるので通常はそのままで。

06結果の読み方（色分け）

結果テーブルは3色で分けられる：

色意味次のアクション
緑 メアド取得成功 営業メール送信対象（※法令確認必須）
黄 問い合わせフォームのみ フォームから送信
赤 どちらも取れず URL見直し or 手動調査 or 電話

統計カード

結果パネル上部の3つの数字：

  • メアド取得：緑行の件数
  • フォームあり：黄行の件数
  • 取得不可：赤行の件数
メアド欄に ; 区切りで複数アドレスが入ることもあります（例：info@〜; contact@〜）。その時は最も一般的な info@ 系を優先して使うといい。

07ダウンロード方法

完了後、「📥 CSV」「📊 Excel」ボタンが有効化される。

どっちを選ぶか

形式特徴こんな時
CSV テキストファイル。軽い Google Sheetsに貼る / 他ツールに取り込む
Excel 色分け装飾付き。読みやすい そのまま自分で見る / 共有する

保存場所

ダウンロードしたファイルは、ブラウザのデフォルトDL先（通常は ダウンロード フォルダ）に入ります。

また、同じファイルがアプリの data/results/ フォルダ内にジョブID付きで自動保存されている（再ダウンロード用）。

09トラブルシューティング

❌ start.bat を押しても黒いウィンドウがすぐ消える

Python が入ってないか、PATHが通ってない可能性。PowerShellで python --version を打って確認。

❌ 「Address already in use」エラー

すでにアプリが起動中。タスクマネージャーで python.exe を終了してから再起動、またはPCを再起動。

❌ 取得できる件数がやたら少ない

URLが間違ってる可能性。URLの先頭に https:// が付いているか、スペルミスがないか確認。

または、対象サイトがBot対策が強い（スタッフサービス等、大手派遣会社に多い）。この場合は手動で調べるしかない。

❌ メアドが取れたけど、明らかにサンプルっぽい（sample@〜）

アプリ側で自動除外してるつもりだけど、漏れることはある。目視でチェックして怪しいのは削除して。

❌ 途中で止まった

黒いウィンドウにエラーが出てる可能性。そのスクショを涼晴 or 開発者に送って。

❌ ブラウザで localhost:5000 開いても「このサイトに到達できません」

黒いウィンドウがまだ起動中じゃない可能性。数秒待ってから再度アクセス。それでもダメなら start.bat もう一回。

10法律と倫理（送信前に絶対読む）

このアプリはメアドを「取得」するだけ。「送信」は別問題です。
取得したメアドに営業メールを送る前に、以下を必ず確認してください。

① 特定電子メール法（特電法）

日本国内では、営業メール送信には原則として事前の同意が必要です。

同意なしで送れるのは、主に：

  • 相手が名刺等で自分のメアドを自ら通知してきた場合
  • 相手が公開webサイト上に送信目的を明記してメアドを公開している場合（← この場合でも、送信拒否の申し出があれば即停止）

単に「ホームページに載ってたから」で送るのは違反の可能性があります。

② 送信時の必須事項

送信するメールには必ず以下を記載：

  • 送信者の氏名・名称
  • 送信者の住所
  • 配信停止方法（リンク or メアド）

③ スクレイピングの利用規約

各社のwebサイトには利用規約があり、一部はスクレイピングを禁止しています。

大量アクセスはサーバー負荷になり、迷惑行為扱いされる可能性もある。このアプリは各社の間隔2秒をデフォルトにしてあるけど、極端に短くしないで。

④ 涼晴ルール

外部メッセージは送信前に自分の目で確認。
Claudeが作った下書きでも、送信ボタンは人間が押す。間違った情報を送ると取り返しつかない。
推奨される使い方：
・すでに取引のある相手 or 名刺交換済みの相手
・問い合わせフォームからの丁寧な初回コンタクト
・展示会・セミナー等での事前同意取得済みの対象
